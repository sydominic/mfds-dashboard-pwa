V20 COLLECT RANGE / DIAGNOSTICS FIX

1. 수집 버튼 날짜 기준 수정
- 빠른수집/기간수집이 화면에 현재 입력된 조회조건(draft)을 직접 사용하도록 수정했습니다.
- 날짜를 바꾼 뒤 조회 버튼을 먼저 누르지 않아도, 수집 버튼이 해당 날짜범위로 바로 동작합니다.
- 수집 후 화면 목록도 같은 날짜범위로 자동 갱신됩니다.

2. MFDS 게시일 파싱 보강
- 게시물 본문 내 여러 날짜가 있을 때 첫 날짜가 아니라 게시물 말미의 실제 게시일/등록일에 가까운 마지막 날짜를 우선 사용합니다.
- 제목/첨부파일명에 과거 날짜가 포함되어 최신 게시물이 기간 필터에서 빠지는 문제를 줄였습니다.
- li/tr 후보를 함께 점검하고 중복은 item_key로 제거합니다.

3. 빠른수집 범위 보강
- 빠른수집도 1페이지만 보지 않고 최대 2페이지까지 확인하도록 변경했습니다.
- 기간수집 페이지 한도를 소폭 상향했습니다.

4. 진단 정보 추가
- 수집 완료 메시지에 대상기간과 수집 확인 최신일을 표시합니다.
- 서버 응답에 게시판별 최신일, 페이지별 후보/매칭 수를 포함합니다.
- /api/collect/last-summary 엔드포인트를 추가했습니다.

5. 로컬 실행 버전검증 수정
- run_local_core.bat의 기대 API 버전이 v20-collect-range-diagnostics로 갱신되었습니다.
- 기존 v18/v19 소스에 남아 있던 v17 기대값 불일치 문제를 수정했습니다.
