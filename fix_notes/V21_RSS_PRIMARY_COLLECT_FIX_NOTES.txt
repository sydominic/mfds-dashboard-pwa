# V21 RSS 우선 수집 수정

## 원인
- 식약처 공식 홈페이지는 게시판 목록 HTML 외에 공식 RSS를 별도로 제공함.
- 기존 수집은 목록 HTML 구조를 직접 파싱하는 방식이라, HTML 구조 변경/접근 실패/파싱 후보 0건 발생 시 최신 게시물이 있어도 확인 0건으로 끝날 수 있었음.
- 실제 식약처 게시판에는 2026-05-28 게시물이 존재하므로, 5/22 이후 게시물이 없는 것이 아니라 수집 경로가 불안정했던 문제임.

## 수정
- 식약처 공식 RSS 주소를 게시판별로 매핑하고 RSS를 1차 수집 경로로 변경.
- RSS 수집 실패 또는 기간수집에서 과거 페이지 확인이 필요한 경우 기존 게시판 HTML 수집을 보조 경로로 사용.
- 빠른수집/기간수집 결과 메시지에 RSS 확인 건수를 표시.
- 수집 결과 0건이면서 오류가 있는 경우 화면에 첫 오류를 표시하도록 수정.
- /api/health에 rssSources 표시.
- /api/collect/last-summary에 boardResults와 errors를 저장하여 원인 확인 가능하도록 보강.

## 확인 방법
1. run_local.bat 실행
2. /api/health에서 apiVersion이 v21-rss-primary-collect-fix인지 확인
3. 최근 7일 또는 2026-05-21~2026-05-28 범위로 빠른수집 실행
4. 메시지에 RSS 확인 건수와 수집 확인 최신일이 표시되는지 확인
