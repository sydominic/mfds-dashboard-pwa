# v8 파서 원인분석 및 수정

## 원인
식약처 목록은 제목, 담당부서, 조회수, 첨부파일, 미리보기/다운받기, 날짜가 한 게시물 안에서 여러 줄로 펼쳐진다.
기존 파서는 같은 DOM block에서 제목과 날짜를 찾는 방식이라 Render 환경/게시판 구조에 따라 최신 게시물을 놓칠 수 있었다.

## 수정
- 기존 tr 파서 유지
- 기존 카드형 li/div 파서 유지
- 신규 text-line fallback 파서 추가
  - 전체 n건 이후 본문 텍스트를 줄 단위로 분해
  - 게시번호/공지사항 marker를 기준으로 게시물 block 분리
  - block 내 첫 정상 제목과 마지막 날짜를 매칭
  - 제목 anchor index를 이용해 view URL 재연결

## 확인
빠른수집 후 [게시판별 수집 결과 보기]에서 보도자료(m_99), 공지(m_74)의 최신게시일을 확인한다.
Render Logs에서 `MFDS m_99 page parse: tr=..., card=..., text=..., deduped=...`를 확인한다.
